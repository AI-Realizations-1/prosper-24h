# Instructions Prosper — Résolution des prompts

## Recherche obligatoire du prompt à exécuter
1. Lire `RETEX/prompts-used.yaml` pour identifier la phase cible (chercher `status: ready`).
2. Ouvrir le prompt dans `.github/prompts/phases/`.
3. Exécuter le prompt.

## Interdiction
- Ne pas dépendre de `/memories/session` comme source principale des prompts de phase.
- Les prompts de phase vivent dans `.github/prompts/phases/`.

## Fallback si prompt introuvable
1. Lire `RETEX/prompts-used.yaml`
2. Lire les `RETEX/phase*-*.md` correspondants
3. Si toujours introuvable: arrêter avec message d'erreur explicite indiquant le chemin manquant.

## Patterns impératifs (Prosper)
- Middleware auth: `authMiddleware` (import `../middleware/auth`)
- Accès user dans contrôleur: `req.userId!`
- Accès studyId: `req.params.studyId`
- Routes: `Router({ mergeParams: true })`
- Token frontend: `localStorage.getItem('accessToken')`
- Private static `validateStudyAccess(studyId, userId)` dans chaque service
